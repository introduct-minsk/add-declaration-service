Selle dokumendi eesmärk on kirjeldada tehnilisi keskkonnanõudeid ja anda paigaldusjuhiseid addServiceDeclaration API teenuse paigalduseks ning integreeritud seadistuseks. addServiceDeclaration teenus on välja töötatud vastavalt addServiceDeclaration API teenuse nõuetele ja spetsifikatsioonidele, mis on kirjeldatud RIA Github-is.

4.       Koormusjaotur

Vastavalt nõuetele on rakendus disainitud mikroteenusena, pidades silmas stateless arhitektuuri kui üht disaini alustõdedest. Seetõttu on API teenus võimeline jooksma N arvu õlal, täiesti sõltumata, ning koormusjaotur EI PEA jälgima stateful loogikat koormuse jaotusel.

Kõiki mikroteenuste eksemplare saab koormustasakaalus hoida, pidamata silmas seansi püsivust. Seega koormusjaoturiga paigaldatava vaikekonfiguratsiooni puhul kasutatakse taotluste jaotamiseks mikroteenuste koopiate vahel lihtsat round-robini koormuse jaotuse printsiipi.

Active / Active koormuse jaotus on sätestatud mikroteenuse addServiceDeclaration keskmes. Dockeri sisevõrgu funktsioone kasutatakse suhtluseks ja koormuse tasakaalustamiseks API teenuse eksemplaride (mikroteenuse konteinerite koopiate) ja andmebaasi vahel.

 

Oluline: antud disaini puhul võib andmebaasi ja koormusjaoturi pidada SPoF-iks, sest paigaldusel on nad ainsad komponendid, mis esinevad ühes eksemplaris. See on tehtud teadlikult kahel eesmärgil: a) ressursside kokkuhoiuks, b) paigalduse lihtsustamiseks.

API teenus on disainitud kujul, mis võimaldab väga lihtsal viisil API teenuse üleviimist andmebaasi connection-pool loogikale. Samamoodi paigaldatavad andmebaasi ja koormusjaoturi konfiguratsiooni failid võivad väga lihtsal viisil olla modifitseeritud, et toetada klasterdatud andmebaasi ja koormusjaoturi paigaldust.

Siiski, kuna antud juhul on tegu prooviülesandega ja me ei sea eesmärki teha paigaldusjuhendit keerulisemaks, siis oleme teadlikult jätnud need võimalused kasutamata. Aga toodangu paigalduse puhul, me kindlasti soovitaksime kasutada klasterdatud andmebaasi ja koormusjaoturi paigaldisi. Rakenduse kood on loodud selle põhimõttega keskmes ja nagu ülalpool mainitud, siis saab see lihtsal kujul, vaid konfiguratsiooni tasemel, olla modifitseeritud seda laadi paigalduse toetamiseks.

 

5.       Eeltingimused paigalduseks

1.    Lähtekood koos Docker compose juhiste failiga on lahti pakitud kettale.

 

2.    Paigalduseks kasutatava serveri operatsiooni süsteem on Ubuntu 18.04 (või kõrgem) ja jookseb x86 platvormil. (Alternatiivsed platvormid on toetatud, kuid antud konkreetne paigaldusjuhend on kirjutatud Ubuntu 18.04 ja x86 platvormile, mis on pakkuja “educated guess” kliendi platvormist)

2.1. Uuendage oma olemasolevat pakettide teeki:

·         sudo apt update

2.2. Installige mõned eeltingimuslikud paketid, mis võimaldavad apt-il pakette HTTPS-is kasutada:

·         sudo apt install apt-transport-https ca-certificates curl software-properties-common

2.3. Lisage oma süsteemi GPG-võti ametliku Dockeri hoidla jaoks:

·         curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

2.4. Lisage Dockeri hoidla APT-i allikate hulka:

·         sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu bionic stable"

2.5. Uuendatud repost värskendage pakettide andmebaasi Dockeri pakettidega:

·         sudo apt update

2.6. Veenduge, et installite vaikimisi Ubuntu repo asemel Dockeri repost:

·         apt-cache policy docker-ce

2.7. Eelmise punkti täitmiseks kontrollige selle väljundit. Väljund peaks välja nägema sarnaselt allolevaga, kuigi Dockeri versiooninumber võib olla erinev:

docker-ce:

  Installed: (none)

  Candidate: 18.03.1~ce~3-0~ubuntu

  Version table:

     18.03.1~ce~3-0~ubuntu 500

        500 https://download.docker.com/linux/ubuntu bionic/stable amd64 Packages

 

3.    Serveris installeeritud Docker Compose versioon peab olema vähemalt 1.25.4 (või kõrgem):

3.1. Installi Docker:

·         sudo apt-get install docker-ce

3.2. Dockeri compose-i skriptide paigaldus:

·         sudo curl -L "https://github.com/docker/compose/releases/download/1.25.4/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

·         sudo chmod +x /usr/local/bin/docker-compose

3.3. Installi openjdk 11 (et luua mikroteenuseid):

·         sudo apt-get install openjdk-11-jdk

3.4. Installi git (lähtekoodi saamiseks repost):

·         sudo apt-get install git

Seadistage see.

 

3.    Vaikeseadete ja ajavööndi konfiguratsioon asub failis: add-declaration-service/src/main/resources/application.yml, vaadake üle selle sobivus.

 

Tähelepanu: Selles failis on kirjeldatud tõlkekeelde nõuded, mis on vaikimisi nõutud kõikide deklaratsioonide tõlgitavate stringide puhul (juhul, kui deklarandi kohta andmebaasis ei ole seadistatud spetsiifilisi reegleid nõutavate tõlgete kohta). Palun vaadake see sektsioon põhjalikult üle – sektsioonis kirjeldatud keeltes tõlked saavad olema kohustuslikud kõikidele deklarantidele, kelle kohta andmebaasis puudub spetsiifiline reegel tõlgete osas.

 

Näide:

 

default-locale:

  locales:

    - language: en

      country: US

    - language: et

      country: EE

 

Tähendab, et kõikidel deklarantidelt, kelle kohta andmebaasis puudub spetsiifiline reegel, nõutakse tõlgitavate stringide puhul eesti- ja ingliskeelseid tõlkeid v.a. väljadel (nt. technicalDescription), kus on spetsiifiliselt nõutud, et tõlgete olemasolu ei ole kohustuslik. (vt. testimisjuhendit täiendava info saamiseks)

 

7.       Paigaldusprotsess

Täitke järgmised sammud addServiceDeclaration API teenuse paigaldamiseks.

Käske tuleb anda, olles lahti pakitud lähtekoodi kataloogi kõrgeimal tasemel.

1.          Koostage jar paketid kõigi rakenduse mikroteenuste jaoks:

·         ./mvnw clean

·         ./mvnw -am -pl :add-declaration-service package -DskipTests

2.          Koostage Dockeri kujutised kõigi rakenduse mikroteenuste jaoks:

·         sudo docker build --build-arg JAR_FILE=services/add-declaration-service/target/add-declaration-service.jar -t add-declaration-service .

3.          Käivitage installatsioon:

·         sudo docker-compose -p consent-service -f docker-compose-dev.yml up -d --scale add-declaration-service=2

 

Tähelepanu! Viimases käsus add-declaration-service=2 – parameter kirjeldab API teenuse duplikaatide arvu, mida lisatakse koormusjaoturi pool-i. Number võib olla nii suur, kui on soovitav konteineri duplikaatide arv (koormusjaotus jagab koormust konteinerite vahel ühtselt).

 

8.       Ühiktestide kävitus

Kuigi, see ei ole otseselt paigaldusprotsessis nõutud, kui on soovi käivitada ühikteste ja veenduda rakenduse kõlblikkuses koodi poolest, saate seda teha, paigaldusprotsessi jooksul käivitades käsku:

·         ./mvnw test

9.       Installimise kontroll

Kui paigaldus on lõpule jõudnud, siis nõuetekohase paigalduse tagamiseks kontrollige allasuva käsuga, et kas kõik konteinerid on üles tulnud ning töötavad korrapäraselt:

·         sudo docker ps -a.

 

10.    

Avage paigaldamiseks kasutatud serveri esilehe IP ning proovige mõnda testimisjuhendist leitavat testijuhtumit katsetamiseks.
